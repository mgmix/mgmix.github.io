# GraphQL 이란?

GraphQL은 페이스북에서 개발된 쿼리 언어이자 API 플랫폼입니다. 기존의 REST API와 비교했을 때, GraphQL은 클라이언트와 서버 간의 데이터 통신을 더 효율적으로 할 수있게 도와줍니다.

어떤 기술이든 생겨난 배경에는 기존 기술의 문제점에서 개선을 위해서 나타난다고 생각합니다. REST API 방식과 비교하여 알아 보겠습니다.

# REST API vs GraphQL

REST API 는 클라이언트가 서버에게 데이터를 요청할 때마다 서버는 사전에 정의된 Endpoint 에 해당되는 데이터를 응답으로 전송합니다. 서버에서는 클라이언트에게 필요한 데이터의 구조화 형식을 미리 정의해야 하죠. 그렇게 정의된 데이터를 클라이언트는 받게 됩니다. 이렇게 메소드와 URI 를 조합해서 예측이 가능한 일정한 정보를 요청할 수 있다는 것이 장점입니다.

하지만 급변하는 기획과 개발 환경 속에서 이렇게 미리 정의한 구조는 변경해야할 일이 잦을 수 있고 호환성을 고려하여 API 별로 Version 관리가 필요 할 수 있습니다. 게다가 ==추가적인 정보를 위해서는 별도의 API 를 호출 해야할 수 있고 클라이언트에서 여러번 요청을 보내는 것은 코드의 비동기 처리를 복잡하게 만들 뿐더러 N+1 쿼리 문제를 발생 시킬 수있습니다.==

GraphQL 은 이러한 문제를 극복하기 위해 원하는 정확한 리소스와 원하는 필드를 지정 할 수있습니다. REST API 에서는 API 작성자, 즉 서버가 리소스를 정의하여 무엇을 포함 할지 결정했다면 GraphQL 은 클라이언트가 무엇을 포함할 지 결정 합니다. REST 에서 Endpoint 를 여러개 만들어서 호출하는 방식과 다르게 단일 진입점으로 단 하나의 Endpoint 만으로 모든 데이터를 요청 할 수 있습니다.

이를 통해 클라이언트는 필요한 데이터만을 취할 수 있고, 네트워크 트래픽이나 대역폭을 절약할 수 있습니다. GraphQL 은 Schema 를 사용하여 Schema 에 정의된 관계에 따라 추가 데이터를 가져오는 복잡한 쿼리를 보낼 수 있습니다. REST API 에서 추가 데이터를 가져오기 위해 여러번의 API 요청을 했던 것을 하나의 요청으로 효율적으로 처리 할 수 있게 됩니다.

REST API 나 GraphQL 이나 모두 HTTP 를 사용하여 요청을 하고 JSON 형태로 데이터를 반환 할 수 있습니다. 단지 정의된 값을 자판기처럼 받을 것이냐, 직접 필요한 값을 요청하여 받을 것이냐 가 주요 차이점 입니다.

GraphQL 도 단점은 존재 합니다. 클라이언트와 서버 측에서 사용하기 위해서는 GraphQL 을 지원하는 도구가 필요합니다. 반면에 REST API 는 다른 사람의 API 를 사용하기 위해 특별한 라이브러리가 필요하지 않죠. REST 에서는 cURL 같은 일반적인 도구나 웹브라우저를 통해서 간단하게 요청을 해 볼 수 있습니다.

GraphQL 은 캐시하기가 어려운 문제도 가지고 있습니다. REST 는 브라우저,CDN, 웹서버 등에서 활용되는 정의된 캐싱 동작이 있는데, HTTP 에서 제공하는 캐싱 전략을 그대로 사용할 수 없기에 GraphQL 만의 캐싱 전략으로 처리해야합니다.

비교적 REST 에 비해서 제어할 수 있는 권한이 커진만큼 리스크도 있습니다. 필요한 데이터만을 요청할 수 있는 유연성이 제공되지만, 데이터 베이스에 영향을 줄 수 있습니다. 여러 테이블을 거치는 쿼리가 요청될 경우 예상치 못한 테이블 스캔이 발생될 수 있죠. 만약 전체 테이블 스캔을 하게 된다면 성능 저하로 이어질 수 있습니다.

# GraphQL 의 특징

# Mutation (데이터 수정)

GraphQL 에서 데이터를 수정 하기 위해서는 Mutation 을 사용해야합니다.

1
2
3
4
5
6
7

mutation AddNewUser($name: String!, $email: String!) {
  addUser(name: $name, email: $email) {
    id
    name
    email
  }
}

위 코드에서 AddNewUser는 해당 Mutation의 이름을 나타냅니다. addUser는 실제로 데이터를 수정하는GraphQL 서버의 리졸버 함수를 호출합니다. 이 함수는 name과 email을 인자로 받아 사용자를 추가하고, 추가된 사용자의 id, name, email을 반환합니다.

Mutation은 데이터베이스에 새로운 데이터를 추가하거나 기존 데이터를 업데이트하고자 할 때 사용됩니다. 이를 통해 클라이언트는 GraphQL 서버를 통해 데이터를 변경할 수 있습니다.

# Subscription (실시간 데이터 알림)

Subscription은 클라이언트가 실시간으로 데이터 변경 사항을 받을 수 있도록 하는 GraphQL의 기능입니다.

1
2
3
4
5
6
7
8

subscription NewPost {
  postAdded {
    id
    title
    content
    author
  }
}

위 코드에서 NewPost는 해당 Subscription의 이름을 나타냅니다. postAdded는 서버 측에서 데이터 변경 사항을 감지하고, 변경된 데이터를 반환하는 GraphQL 서버의 리졸버 함수를 호출합니다. 클라이언트는 해당 Subscription을 구독(subscribe)하여 새로운 게시물이 추가될 때마다 id, title, content, author와 같은 필드를 받게 됩니다.

Subscription은 실시간 채팅, 실시간 알림, 실시간 데이터 스트리밍 등 실시간 기능을 구현할 때 유용합니다. 클라이언트는 GraphQL 서버와 WebSocket 연결을 유지하여 서버에서 발생하는 데이터 변경 사항을 실시간으로 받을 수 있습니다.
